# 位置环改进记录

## 背景

原位置闭环采用“位置环 -> 速度环 -> 电流环”的三级级联结构。为了减少链路长度、提升响应速度，并参考 VESC 位置模式的实现思路，现将位置控制改为“位置环 -> 电流环”两级结构，并对位置环的微分与调度方式做进一步优化。

---

## 本次改进内容

### 1. 位置环改为直接输出 `q` 轴目标电流

原结构：

```text
位置环 -> 目标速度 -> 速度环 -> 目标 iq -> 电流环
```

现结构：

```text
位置环 -> 目标 iq -> 电流环
```

### 改进目的

- 减少一个中间环节，缩短控制链路。
- 降低速度环引入的额外相位滞后。
- 提高位置响应速度。
- 使位置环更接近伺服位置控制的“位置误差直接决定转矩”思路。

### 当前实现位置

- `User/foc/loop_control.c`
- `loopControl_run_positionLoop()`

---

### 2. 位置环由 PI 改为 PD

位置环初始化已改为 `PID_MODE_PID`，但 `KI = 0`，因此当前等效为 PD 控制。

相关参数位于：

- `User/app/user_config.h`

当前参数：

```c
#define POSITION_PID_KP           0.5f
#define POSITION_PID_KI           0.0f
#define POSITION_PID_KD           0.02f
#define POSITION_PID_OUT_MIN      -0.8f
#define POSITION_PID_OUT_MAX      0.8f
```

### 改进目的

- 保留比例项用于快速建立回复转矩。
- 使用微分项提供阻尼，减小超调与振荡。
- 位置环不引入积分项，避免静差积分在小范围抖动时累积。

---

### 3. D 项改为测量微分，避免目标阶跃造成 D kick

原始常规 PID 微分项形式：

```text
D = d(error) / dt
```

当目标位置突变时，误差会瞬间跳变，导致 D 项出现尖峰，即所谓 **D kick**。

现改为测量微分：

```text
D = -d(position_feedback) / dt
```

在代码中的等效形式为：

```c
position_speed_rad_s = (position_rad - prev_position_rad) / position_loop_dt;
pid->derivative = -position_speed_rad_s;
pid->d_term = pid->kd * pid->derivative;
```

### 改进目的

- 避免目标位置阶跃时 D 项冲击。
- 让微分项只对“实际运动速度”产生阻尼作用。
- 改善位置指令跳变时的电流尖峰问题。

### 结果

位置环 D 项现在本质上更接近：

```text
阻尼项 = -K_d * 机械角速度
```

这与很多伺服系统中的速度阻尼思想一致。

---

### 4. 位置环分频改为相对电流环

早期改法中，位置环分频曾临时写成：

```text
位置环分频 = 速度环分频 × 位置环分频
```

后来已改为：

```text
位置环分频仅相对电流环
```

即位置环执行周期为：

$$
T_{pos} = T_{current} \times position\_loop\_divider
$$

对应实现：

```c
uint16_t position_total_divider = (uint16_t)position_loop_divider;
```

### 改进目的

- 语义更清晰：位置环直接相对电流环调度。
- 避免位置环频率被速度环分频间接影响。
- 更方便后续单独调整位置环频率。

---

### 5. 删除位置环接口中的无用 `speed_loop_divider` 参数

在位置环不再级联速度环之后，`loopControl_run_positionLoop()` 中的 `speed_loop_divider` 参数已经没有实际作用，因此已删除。

#### 旧接口

```c
loopControl_run_positionLoop(..., uint8_t speed_loop_divider, uint8_t position_loop_divider)
```

#### 新接口

```c
loopControl_run_positionLoop(..., uint8_t position_loop_divider)
```

同步修改位置：

- `User/foc/foc.h`
- `User/foc/loop_control.c`
- `User/motor/position_closed.c`

### 改进目的

- 去掉无效参数，减少歧义。
- 保持接口和实际控制结构一致。

---

## 当前控制逻辑总结

当前的位置闭环逻辑如下：

```text
目标机械位置 - 当前机械位置
        ↓
      PD控制
   (P + 测量微分D)
        ↓
    目标 q轴电流 iq
        ↓
      电流环 PI
        ↓
      电压矢量输出
```

其中：

- `P` 决定位置误差对应的回复力矩。
- `D` 决定对实际运动速度的阻尼抑制。
- 电流环负责把 `iq` 精确变成电机实际转矩。

---

## 当前实现的优点

1. **控制链更短**
   - 取消速度环级联后，位置到转矩的传递更直接。

2. **响应更快**
   - 减少了中间速度环动态。

3. **更适合小角度快速定位**
   - 尤其适合位置保持、角度跟踪类应用。

4. **避免 D kick**
   - 目标位置跳变时不再因为误差微分造成尖峰。

5. **参数物理意义更直观**
   - `POSITION_PID_KP`：每 rad 误差对应多少 A 的转矩电流。
   - `POSITION_PID_KD`：每 rad/s 实际速度对应多少 A 的阻尼电流。

## 当前仍可继续优化的方向

### 1. 加入小误差增益衰减

现已加入小误差增益衰减参数：

```c
#define POSITION_PID_GAIN_DEC_ERROR_RAD  0.05f
```

当位置误差满足：

```text
|error| < POSITION_PID_GAIN_DEC_ERROR_RAD
```

时，位置环的 `kp / ki / kd` 会按误差比例线性缩小。

当前到位判定仍保留死区：

```c
if (fabsf(position_error) <= POSITION_DEADBAND_RAD)
```

这样做的目的：

- 接近目标位置时自动降低位置环刚度。
- 减少死区边缘附近的电流突变。
- 降低贴近目标时的振荡与抖动。

### 2. 给测量微分增加低通滤波

现已加入位置微分低通滤波参数：

```c
#define POSITION_PID_D_FILTER_ALPHA      0.2f
```

实现形式为：

```text
speed_filtered += alpha * (speed_raw - speed_filtered)
```

虽然已经改为测量微分，但如果编码器反馈有量化噪声或抖动，D 项仍可能放大高频噪声。

加入该滤波后：

```text
position_speed_filtered = LPF(position_speed_rad_s)
```

再用于 D 项计算，可减轻编码器量化噪声对阻尼项的放大。

### 3. 根据负载情况评估是否需要积分项

若系统存在明显静摩擦或恒定外负载，在纯 PD 下可能会有少量稳态误差。此时可以考虑：

- 继续保持 PD；
- 或仅在小误差区引入极弱积分；
- 或改为重力/摩擦前馈补偿。

## 备注

本次改动思路参考了 VESC 位置模式的若干工程经验，但当前实现仍保留本项目自身特点：

- 使用**机械多圈位置**作为位置反馈，而不是单圈角度误差。
- 位置环输出直接使用**物理电流单位 A**，而不是归一化输出。
- 更适合当前项目的机械定位和角度保持应用。
